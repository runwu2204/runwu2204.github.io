il2cpp逆向

一个Unity游戏保护方案的分析和还原符号信息，偷学对global-metadata保护的思路 - 知乎 (zhihu.com)

关注方向

windows

关注当前文件的GameAssembly.dll(主要函数)和xxxx_Data\il2cpp_data\Metadata\global-metadata.dat(存储符号表以及一些引用)

现在主要重点集中于加密global-metadata.dat用于防止通过Il2CppDumper 逆向出Assembly-CSharp.dll(存储主要逻辑，在il2cpp中用于存放cpp编译为dll的函数调用地址)

android

关注当前文件的.\lib\armeabi-v8a\libil2cpp.so(主要函数)和.\assets\bin\Data\Managed\Metadata\global-metadata.dat(存储符号表以及一些引用)

现在主要重点集中于加密global-metadata.dat用于防止通过Il2CppDumper 逆向出Assembly-CSharp.dll(存储主要逻辑，在il2cpp中用于存放cpp编译为dll的函数调用地址)

解密global-metadata.dat

global-metadata.dat解密基本都基于 程序运行时，需要将global-metadata.dat解密

1、通过dump memory

2、通过逆向加载global-metadata.dat的逻辑

1、搜索对应函数

可通过搜索字符串global-metadata.dat或者Metadata来寻找对应逻辑(程序加载dat时需要对应的路径)

也可通过查询导入表的MapViewOfFile,因为读取文件时，源码使用的就是这个函数，只不过这个函数使用处较多，需要仔细查找

2、找到加载逻辑

一般形式如下，如果有加密，注意对data的操作

3、还原对应global-metadata.dat

注意header位置(0到0x110)

前两个4字节数据为magic number和version

其他都为一类数据的offset和count,注意这类数据是否超过实际文件范围

3、通过Il2CppDumper还原逻辑

windows il2CppDumper.exe ".\GameAssembly.dll" ".\xxxx_Data\il2cpp_data\Metadata\global-metadata2.dat" ".\"

android il2CppDumper.exe ".\lib\armeabi-v8a\libil2cpp.so" ".\assets\bin\Data\Managed\Metadata\global-metadata.dat".\

执行成功后，dump下的目录中script.json保存符号信息

windows下的GameAssembly.dllandroid下的libil2cpp.so也存放着函数的具体实现(因为il2cpp本身就是cs转cpp库的操作)可通过 Il2CppDumper目录下的ida_py3.py加载script.json导入对应符号

DummyDll\Assembly-CSharp.dll存放着函数信息

可通过cs逆向工具逆向出对应的VA

VA可通过ida进行跳转查找对应的逻辑
